  <name>Task 1: Fix DOMPurify namespace import in RichText.js</name>
  <files>src/blocks/RichText.js</files>
  <action>
    Replace the default import of dompurify with a namespace import and add defensive resolution.

    Current line 19:
      import DOMPurify from 'dompurify'

    Replace with:
      import * as DOMPurifyModule from 'dompurify'
      const DOMPurify = DOMPurifyModule.default ?? DOMPurifyModule

    The rest of the file is unchanged — DOMPurify.sanitize() on line 28 continues to work because
    `DOMPurify` is now the resolved module (either the .default export or the namespace itself).

    Root cause: TSUP CJS interop wraps the default import with `_interopDefault`, producing
    `DOMPurify__default.default.sanitize` in the output — a double .default access that fails under
    Next.js/turbopack ESM interop. The namespace import with defensive resolution bypasses this chain.

    Note: Tests mock 'dompurify' with `mock.module` — they pass regardless of import style. This fix
    targets the runtime dist output only. Do not change the test files.
  </action>
