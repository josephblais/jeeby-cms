<decisions_made>

- Replaced isomorphic-dompurify with dompurify (browser-only) — isomorphic-dompurify v3 uses jsdom v26 which has a transitive @exodus/bytes ESM dependency that breaks CJS require() in Node 22. Since the whole package is "use client", dompurify (no jsdom) is the correct choice.
- Files changed: src/blocks/RichText.js, src/blocks/RichText.test.js, src/blocks/index.test.js, package.json, tsup.config.js
- Blocks data shape: { blocks: [...] } — not a plain array. getCMSContent returns this shape directly. The test data in the test page needed to be wrapped: const testBlocks = { blocks: [...] }
- npm pack approach (not npm link / file: symlink) required for local dev testing — symlinks cause Node to resolve from library's node_modules, not test app's, causing dependency conflicts
</decisions_made>
