<decisions_made>

- Firestore path structure: flat top-level collections per content type — pages/{slug}, events/{slug} etc. One Firebase project per site (not multi-tenant). Rationale: cleaner security rules, separate billing, no data bleed between sites.
- DOMPurify SSR: typeof window checks unreliable in Next.js (window polyfilled server-side). DOMPurify.sanitize stub exists on server but returns input unchanged. Fix: useState initialized to stripDangerous(raw), useEffect upgrades to DOMPurify after mount.
- Blocks API: className on Blocks = outer container div. blockClassName = each individual Block wrapper div.
- npm install workflow for tgz file: references: must use npm install --force + npm cache clean --force + delete node_modules + delete package-lock.json. Direct folder file: references don't work (Next.js doesn't follow symlinks). .next/ cache must also be cleared after reinstall.
- Admin SDK env vars: FIREBASE_ADMIN_PROJECT_ID, FIREBASE_ADMIN_CLIENT_EMAIL, FIREBASE_ADMIN_PRIVATE_KEY (private key needs .replace(/\\n/g, '\n') — already handled in admin.js)
- stripDangerous is defense-in-depth only — TODO in RichText.js to sanitize HTML on admin input before writing to Firestore

</decisions_made>
