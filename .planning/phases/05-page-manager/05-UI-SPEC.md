## Interaction Contracts

### Page List Table (PageManager)

Layout: full-width table, four columns.

| Column | Header | Content | Width |
|--------|--------|---------|-------|
| 1 | Name | Inline-editable display name; shows input on click/edit-icon click | 30% |
| 2 | Slug | Inline-editable slug; shows input on click/edit-icon click | 35% |
| 3 | Last Published | Formatted date or "Never" | 20% |
| 4 | Actions | "Delete" button (text, destructive color) | 15% |

Semantic HTML: `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` for all column headers, `<td>` for data cells. Do not use `<div>` grid for the table layout.

"New Page" button: top-right above the table. Uses `.jeeby-cms-btn-primary` class, accent background (#2563EB), white text, minHeight 44px, borderRadius 4px.

Table section heading: `<h2>Pages</h2>` (20px, semibold) sits above the New Page button row — provides page landmark and heading hierarchy (AdminNav has no visible H1 in the shell).

### Empty State

When `listPages` returns zero results:

```
[centered container, padding 48px vertical]
  "No pages yet."                  [h2, 20px, 600, #374151]
  "Create your first page."        [p, 14px, 400, #6B7280]
  [New Page button — same as table header button]
```

The empty state `<h2>` replaces the table `<h2 Pages>` — only one `<h2>` visible at a time.

### Create Page Modal

Trigger: "New Page" button. On click: open CreatePageModal, move focus to first field (display name input).

Modal shell:
```
<div role="dialog" aria-modal="true" aria-labelledby="create-modal-heading">
  <h2 id="create-modal-heading">Create New Page</h2>
  [backdrop: full-screen overlay, background rgba(0,0,0,0.5)]
  [card: maxWidth 480px, padding 32px, background #fff, borderRadius 8px]
```

Fields (in order, all required):
1. Display Name — `<label for="cms-page-name">Page name</label>` + `<input id="cms-page-name" type="text">`
2. Slug — `<label for="cms-page-slug">Slug</label>` + `<input id="cms-page-slug" type="text" aria-describedby="cms-slug-error cms-slug-hint">` + hint text below: "e.g. /blog/my-post"
3. Template — `<label for="cms-page-template">Template</label>` + `<select id="cms-page-template">` — HIDDEN entirely when no templates are registered (do not render an empty select)

Slug validation error: renders below slug field as `<p id="cms-slug-error" role="alert">`. Hidden when no error.

Buttons (right-aligned row, 16px gap):
- Discard: `.jeeby-cms-btn-ghost` — background none, border `1px solid #E5E7EB`, minHeight 44px, padding `8px 16px`
- Create Page: `.jeeby-cms-btn-primary` — accent background, white text, minHeight 44px, padding `8px 24px`; `disabled` + `aria-busy="true"` + `opacity: 0.6` while saving

On success: modal closes, focus returns to "New Page" trigger button, new row appears at top of list. Live region announces: "Page created successfully."

On close (Discard or Escape): focus returns to "New Page" trigger button.

### Delete Confirmation Modal

Trigger: "Delete" button in table row Actions column.

Modal heading: `<h2 id="delete-modal-heading">Delete page?</h2>`

Body copy: "Delete [slug]? This cannot be undone." where [slug] is the actual slug value, e.g. "Delete /blog/my-post? This cannot be undone."

Buttons (right-aligned, 16px gap):
- Keep Page: `.jeeby-cms-btn-ghost` — same style as create modal Discard button
- Delete Page: `.jeeby-cms-btn-destructive` — background #DC2626, white text, minHeight 44px, padding `8px 24px`; `disabled` + `aria-busy="true"` + `opacity: 0.6` while deleting

On confirm: modal closes, row removed from list, focus returns to the next Delete button in the list (or "New Page" button if the list is now empty). Live region announces: "Page deleted."

On close (Keep Page or Escape): focus returns to the Delete button that opened the modal.

### Inline Edit (Name and Slug cells)

Trigger: click anywhere on a Name cell or Slug cell (entire cell is the click target), OR click the edit icon button adjacent to the cell value.

Edit icon button: `<button type="button" aria-label="Edit name for [page name]">` — text "Edit" or a pencil symbol (Unicode \u270F). Hidden visually until row is hovered (CSS only — always in DOM for keyboard users, `opacity: 0` on non-hover, `opacity: 1` on focus/hover).

Input state:
```
<input
  type="text"
  value={editValue}
  aria-label="Edit slug for [page name]"
  aria-describedby="cms-rename-error-[slug]"
/>
```

Keyboard: Enter saves, Escape cancels and restores original value.
Blur saves (no separate Save button — avoids blur-before-click race condition per RESEARCH.md Pitfall 7).

Error: `<p id="cms-rename-error-[slug]" role="alert">` renders below the row, full colspan. Hidden when no error.

Debounce: 300ms after last keystroke before running `validateSlug()`. Validation fires on debounce tick; save fires on Enter/blur (not gated on debounce).

### Loading and Async States

Page list loading: while `listPages` is in flight, show a centered spinner inside the `<main>` area. Same spinner pattern as AdminPanel (`32px`, `border: 3px solid #2563EB`, `borderTopColor: transparent`, `borderRadius: 50%`). Accessible: `role="status" aria-label="Loading pages"`.

After any mutation (create, delete, rename): re-fetch `listPages` to reflect server state. Show brief loading state during re-fetch.

## Accessibility Contract

These obligations are in addition to CLAUDE.md baseline and must be verified by source inspection tests.

### Modals (CreatePageModal, DeletePageModal)

- `role="dialog"` on the container div
- `aria-modal="true"` on the container div
- `aria-labelledby` pointing to the modal's `<h2>` id
- Focus moves to the first focusable element inside the dialog on open
- Focus is trapped inside: Tab and Shift+Tab cycle only within the dialog's focusable elements
- Escape key closes the dialog
- Focus returns to the trigger button on close
- Backdrop click does NOT close the modal (prevents accidental dismissal — CONTEXT.md specifies explicit Discard/Keep Page/Escape only)

Focus trap implementation: use `useRef` to query `'button, input, select, textarea, [tabindex]:not([tabindex="-1"])'` within the dialog ref. Cycle on Tab/Shift+Tab with `onKeyDown`.

### Forms

- Every `<input>` and `<select>` has an explicit `<label>` with matching `for`/`id` pair
- Slug field has `aria-describedby` pointing to both the hint text id and the error text id
- Error messages use `role="alert"` so they are announced immediately without user action
- Required fields do NOT use `aria-required` alone — the `required` attribute on the input suffices (browser validation is `noValidate`-disabled; custom validation handles it)
- Submit buttons are `type="submit"`, cancel buttons are `type="button"`

### Table

- Use `<table>` with `<thead>`, `<tbody>`, `<th scope="col">` for all column headers
- Table has an accessible name via the preceding `<h2>` — do not use `aria-label` on the table itself (h2 provides context)
- Each Actions cell Delete button: `aria-label="Delete [slug]"` so screen readers announce which page will be deleted (not just "Delete")
- Each edit icon button: `aria-label="Edit name for [name]"` or `"Edit slug for [name]"`

### Live Regions

A single `<div aria-live="polite" aria-atomic="true">` sits at the top of the PageManager, visually hidden (position absolute, clip), used for all success announcements:
- "Page created successfully."
- "Page deleted."
- "Page renamed successfully."
- "Rename failed. Please try again." (on delete-step failure in renamePage)

Error announcements use `role="alert"` on the inline error elements (already assertive — no separate live region needed).

### Keyboard Navigation

- All interactive elements (buttons, inputs, select) are reachable by Tab in DOM order
- "New Page" button is the first interactive element after the `<h2>` heading in normal flow
- Table rows do NOT have `tabIndex` — only the interactive elements inside them are focusable
- Inline edit: after committing a rename (Enter), focus stays on the cell's edit trigger button (not lost)
