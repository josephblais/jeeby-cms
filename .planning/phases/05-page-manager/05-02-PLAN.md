   function formatDate(ts) {

   }

3. EXPORTED COMPONENT `PageManager`:
   - Call `useCMSFirebase()` to get `{ db, templates }`
   - State: `pages` (array), `loading` (boolean, starts true), `error` (string|null), `announcement` (string for live region)
   - State for inline edit: `editingSlug` (slug being edited or null), `editField` ('name'|'slug'|null), `editValue` (current input value), `editError` (string|null)
   - Ref: `debounceRef` for slug validation debounce

4. LOAD PAGES on mount:
   ```javascript
   const loadPages = useCallback(async () => {
     setLoading(true)
     try {
       const result = await listPages(db)
       setPages(result)
     } catch (err) {
       setError('Failed to load pages.')
     } finally {
       setLoading(false)
     }
   }, [db])

   useEffect(() => { loadPages() }, [loadPages])
   ```

5. INLINE EDIT HANDLERS:
   - `startEdit(slug, field)` — sets editingSlug, editField, editValue to current value
   - `cancelEdit()` — clears editingSlug, editField, editValue, editError
   - `handleEditChange(val)` — sets editValue; if field is 'slug', debounce 300ms then run validateSlug against the page's template pattern
   - `commitEdit()` — on Enter or blur:
     - If field is 'name': call `savePage(db, editingSlug, { name: editValue })`, reload pages, announce "Page renamed successfully."
     - If field is 'slug': validate slug first (not debounced — immediate check), then call `renamePage(db, editingSlug, editValue)`, reload pages, announce "Page renamed successfully."
     - On error: set editError with message. For rename delete-step failure: "Rename failed. The old page may still exist -- check Firestore and try again."
   - `handleEditKeyDown(e)` — Enter calls commitEdit(), Escape calls cancelEdit()
