1. **How to import styles** -- `import 'jeeby-cms/dist/styles.css'` in the consumer's layout or page.

2. **CSS Custom Properties** -- A table listing all consumer-overridable vars with their default values:

| Variable | Default | Purpose |
|----------|---------|---------|
| `--jeeby-cms-accent` | `#4a90d9` | Primary action color (buttons, links, active states) |
| `--jeeby-cms-focus-ring` | `#4a90d9` | Focus indicator color |
| `--jeeby-cms-bg-surface` | `#191919` | Admin panel background |
| `--jeeby-cms-text-primary` | `#e8e6e1` | Main admin UI text color |
| `--jeeby-cms-max-width` | `720px` | Maximum width of content blocks |
| `--jeeby-cms-block-spacing` | `1.5rem` | Vertical spacing between blocks |
| `--jeeby-cms-gallery-columns` | `3` | Number of columns in gallery grid |

3. **How to override** -- Show the CSS selector pattern:
```css
.jeeby-cms-admin {
  --jeeby-cms-accent: #e74c3c;
  --jeeby-cms-bg-surface: #1a1a2e;
}
```

4. **Block styling** -- Explain that content block components (Title, Paragraph, RichText, Image, Video, Gallery) ship with NO visual styles. They inherit from the consumer's site CSS. Consumers apply styles via the `className` prop on each block component or by targeting elements inside their own CSS.

5. **Scoping** -- All admin panel styles are scoped under `.jeeby-cms-admin` and do not affect consumer page content.

Keep the section concise (under 60 lines of markdown). Use plain language per the project's README style.
