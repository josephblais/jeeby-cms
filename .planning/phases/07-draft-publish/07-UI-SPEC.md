## Component Inventory

### 1. EditorHeader (extended)

File: `src/admin/EditorHeader.js`
Change: Add right-slot publish controls.

Right slot layout (left to right, single row, `display: flex`, `alignItems: center`, `gap: 16px`):

```
Last published: Mar 18, 2026  ·  Unpublished changes    Saved ✓    [ Publish ]
```

- "Last published: [date]" — plain `<span>`, 14px body, weight 400
- Date format: `MMM D, YYYY` (e.g. "Mar 18, 2026"); when never published: "Last published: Never"
- Separator `·` — decorative, `aria-hidden="true"`
- "Unpublished changes" — plain `<span>`, 14px body, weight 400; rendered only when `hasDraftChanges === true`; no color, no dot, no badge
- Save status — existing `<div role="status" aria-live>` block, unchanged
- `[ Publish ]` button — `<button type="button" className="jeeby-cms-btn-primary">`, min-height 44px; disabled when modal is open OR publish is in-flight (`aria-busy="true"` when in-flight, `disabled` when modal open)

New props accepted by EditorHeader:
- `lastPublishedAt` — ISO string or null
- `hasDraftChanges` — boolean
- `onPublish` — function (opens confirmation modal)
- `publishStatus` — `'idle' | 'publishing' | 'error'` (controls Publish button disabled/busy state)

Accessibility requirements:
- The entire right slot is a single `<div>` — no role needed
- "Unpublished changes" span: no live region needed (it is static state, not a dynamic announcement)
- Publish button: `aria-disabled="true"` when modal is open (to preserve focus order); use `disabled` only when publish is in-flight to prevent double-submit

### 2. PublishConfirmModal (new)

File: `src/admin/PublishConfirmModal.js`
Pattern: Identical to `CreatePageModal` / `DeletePageModal` — reuse focus trap + Escape + triggerRef.

Structure:
```
role="dialog" aria-modal="true" aria-labelledby="publish-modal-heading"
className="jeeby-cms-modal-backdrop"  →  className="jeeby-cms-modal-card"
  <h2 id="publish-modal-heading">Publish '[Page Name]'?</h2>
  <p>[confirmation body copy]</p>
  [error message — inline, role="alert", only when publishError is set]
  <div buttons row>
    <button className="jeeby-cms-btn-ghost">Cancel</button>
    <button className="jeeby-cms-btn-primary" aria-busy when publishing>Publish now</button>
  </div>
```

Focus management:
- On open: focus the Cancel button (first focusable; safer default — avoids accidental publish on Enter)
- Tab trap: same pattern as DeletePageModal (querySelectorAll focusable, wrap at boundaries)
- Escape: close modal, return focus to Publish button in EditorHeader
- On error: modal stays open; error message rendered inline above the button row with `role="alert"`
- On success: modal closes; focus returns to Publish button (now updated state)

Props: `open`, `pageName`, `onClose`, `onConfirm`, `triggerRef`, `publishing`, `publishError`

Modal card max-width: 480px (matches existing modals).

### 3. PublishToast (new)

File: `src/admin/PublishToast.js`
Pattern: Same as `UndoToast` structure.

Structure:
```
role="status" aria-live="polite" aria-atomic="true"
className="jeeby-cms-publish-toast"
style: position fixed, bottom 24px, left 50%, transform translateX(-50%), zIndex 200
  background: #1f2937, color: #f9fafb
  display flex, alignItems center, gap 12px
  <span>Page published successfully.</span>
```

- No action button (unlike UndoToast which has an Undo button)
- Auto-dismisses after 3 seconds (same pattern as UndoToast lifecycle in PageEditor)
- zIndex: 200 (matches UndoToast; both toasts will not overlap — publish toast fires only after undo toast window has passed)

Accessibility: `role="status"` + `aria-live="polite"` announces to screen reader on mount. Auto-dismiss at 3s does not move focus.

## Interaction States

### Publish button states

| State | Rendered As |
|-------|-------------|
| Idle, no draft changes | `<button>Publish</button>` — enabled, normal |
| Idle, has draft changes | `<button>Publish</button>` — enabled, normal (the "Unpublished changes" text is the only indicator) |
| Modal open | `disabled` attribute set; button is visually disabled |
| Publish in-flight (after Publish now clicked) | `aria-busy="true"`, `disabled`; label: "Publishing…" |
| Publish succeeded | Returns to idle; date updated; "Unpublished changes" disappears |
| Publish failed | Modal stays open; error shown in modal; button returns to enabled |

### Header right slot — state matrix

| `hasDraftChanges` | `lastPublishedAt` | Right slot text |
|-------------------|-------------------|-----------------|
| false | null | `Last published: Never` |
| true | null | `Last published: Never · Unpublished changes` |
| false | ISO string | `Last published: Mar 18, 2026` |
| true | ISO string | `Last published: Mar 18, 2026 · Unpublished changes` |

## Accessibility Contract

### WCAG 2.1 AA requirements for Phase 7 components

1. EditorHeader right slot
   - All text meets 4.5:1 contrast against header background (deferred to Phase 8 full CSS; Phase 7 inherits defaults)
   - Publish button: visible focus ring (browser default is acceptable in Phase 7)
   - Touch target: `minHeight: '44px'` on Publish button
   - "Unpublished changes" is static text, not a live region — it reflects loaded state, not a dynamic change

2. PublishConfirmModal
   - Focus trap: Tab and Shift+Tab cycle within modal only
   - Escape closes modal and returns focus to Publish button
   - `aria-modal="true"` on dialog element
   - `aria-labelledby` pointing to `<h2 id="publish-modal-heading">`
   - Cancel button receives initial focus on open (not Publish now — prevents accidental Enter-to-publish)
   - Error message: `role="alert"` (assertive announcement); displayed inline above button row
   - Both buttons: `minHeight: '44px'`
   - Publish now button: `aria-busy="true"` during in-flight, label changes to "Publishing…"

3. PublishToast
   - `role="status"` + `aria-live="polite"` + `aria-atomic="true"`
   - Auto-dismiss at 3s does not steal focus
   - No interactive elements in this toast (unlike UndoToast) — no keyboard trap risk
   - zIndex 200 ensures it floats above canvas but below modals (modal backdrop is zIndex 1000)

4. Focus return chain
   - Publish button in EditorHeader → opens modal → Cancel/Escape returns focus to Publish button
   - Publish success → modal closes → focus returns to Publish button (now in updated state)

Source: CONTEXT.md §Claude's Discretion (focus trap, ARIA live region), CLAUDE.md WCAG AA requirement, existing modal patterns in DeletePageModal.js and CreatePageModal.js.
