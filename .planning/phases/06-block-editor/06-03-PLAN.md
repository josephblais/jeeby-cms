Note on contentEditable: React warns about children in contentEditable. `suppressContentEditableWarning` suppresses this. The initial text is set via children. Subsequent edits use onInput to push text to parent via onChange. The contentEditable div should only set children on first render. To handle this correctly, use a ref to set textContent only when data.text changes externally (not from user input):

```javascript
const divRef = useRef(null)
const isInternalChange = useRef(false)

function handleInput(e) {
  isInternalChange.current = true
  onChange({ ...data, text: e.currentTarget.textContent })
}

useEffect(() => {
  if (!isInternalChange.current && divRef.current && data?.text !== undefined) {
    if (divRef.current.textContent !== data.text) {
      divRef.current.textContent = data.text
    }
  }
  isInternalChange.current = false
}, [data?.text])
```

Add `ref={divRef}` and `onInput={handleInput}` to the contentEditable div. Remove the children `{data?.text || ''}` -- use the useEffect for initial content too.

Wait -- actually `toEmbedUrl` returns the original URL for unknown platforms (not null). So we need a different check. Looking at Video.js source: it returns the URL as-is for non-matches. The error should show when the URL is entered but does not match YouTube, Vimeo, or Loom patterns. Simplify:

```javascript
const rawUrl = data?.url ?? ''
const embedUrl = rawUrl ? toEmbedUrl(rawUrl) : null
```
