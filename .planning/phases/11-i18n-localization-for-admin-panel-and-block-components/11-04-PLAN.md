<objective>
Deliver the admin-side bilingual editing experience. Create the `LocaleSwitcher` component (a WCAG-compliant tab bar with keyboard navigation), mount it in `PageEditor` above the block canvas, and update all five block editor forms (Title, Text, Image, Video, Gallery) to read and write locale-keyed objects when `isLocalized` is true while preserving exact monolingual behavior when it is false.

Purpose: This plan closes the remaining I18N requirements (09, 10, 11, 12) and delivers the full editor UX for bilingual content management. The critical correctness concern is I18N-12: the spread pattern `{ ...(data.text ?? {}), [locale]: newVal }` guarantees that typing in FR never overwrites EN (and vice versa). D-06 demands that `isLocalized: false` leaves every editor form byte-identical in behavior.
Output: 1 new admin component, 1 PageEditor update, 5 editor form updates.

```jsx
export function LocaleSwitcher() {
  const { isLocalized, locale, setLocale } = useCMSFirebase()
  if (!isLocalized) return null

  function handleKeyDown(e) {
    if (e.key === 'ArrowRight') { /* focus + setLocale('fr') */ }
    if (e.key === 'ArrowLeft')  { /* focus + setLocale('en') */ }
  }

  return (
    <div role="tablist" aria-label="Content language" className="jeeby-cms-locale-switcher">
      <button
        role="tab"
        aria-selected={locale === 'en'}
        tabIndex={locale === 'en' ? 0 : -1}
        onClick={() => setLocale('en')}
        onKeyDown={handleKeyDown}
        type="button"
      >EN</button>
      <button
        role="tab"
        aria-selected={locale === 'fr'}
        tabIndex={locale === 'fr' ? 0 : -1}

        onKeyDown={handleKeyDown}
        type="button"
      >FR</button>
    </div>
  )

```

Editor form read/write pattern (I18N-11, I18N-12):

```js
// Read
const textValue = isLocalized
  ? (data?.text?.[locale] ?? '')
  : (data?.text ?? '')

// Write — the spread before [locale] is MANDATORY to preserve the other locale
function handleChange(newVal) {
  if (isLocalized) {
    onChange({ ...data, text: { ...(data.text ?? {}), [locale]: newVal } })
  } else {
    onChange({ ...data, text: newVal })
  }
}
```

The Plan 01 I18N-12 test asserts the regex `\{\s*\.\.\.[^}]*\[locale\]\s*:` — meaning the code must spread existing object contents BEFORE the computed `[locale]` key. Any implementation that does `{ [locale]: newVal }` without the spread WILL fail the test AND lose the other locale's content.

ARIA tablist pattern (WCAG 2.1.1 + WAI-ARIA Authoring Practices):
- Container: role="tablist", aria-label
- Tabs: role="tab", aria-selected, tabIndex for roving focus
- ArrowRight: focus next tab (wrap around) + call setLocale
- ArrowLeft: focus previous tab + call setLocale
- Only the active tab has tabIndex=0; others have tabIndex=-1 (roving tabindex pattern)

Required hook for context read: `useCMSFirebase` is already exported from `src/index.js`. Import path from admin files: `import { useCMSFirebase } from '../index.js'` (LocaleSwitcher) or `from '../../index.js'` (editor forms).

    ```js
    import { useCMSFirebase } from '../index.js'

    // LocaleSwitcher — bilingual content language tab bar.
    // ACCESSIBILITY: WCAG 4.1.2 (role="tablist" + role="tab" + aria-selected),
    //   2.1.1 (keyboard — Arrow keys swap locales via roving tabindex pattern),
    //   1.3.1 (aria-label provides programmatic context for the tab group).
    //
    // Renders NOTHING when isLocalized is false — D-04 / D-06 guarantee zero UI
    // leak into monolingual mode.
    //
    // Context dependency: useCMSFirebase() must return { isLocalized, locale, setLocale }.
    // Those three keys are added to the context value in Plan 02 (CMSProvider).
    export function LocaleSwitcher() {
      const { isLocalized, locale, setLocale } = useCMSFirebase()
      const enRef = useRef(null)
      const frRef = useRef(null)

      if (!isLocalized) return null

      function handleKeyDown(e) {
        if (e.key === 'ArrowRight') {
          e.preventDefault()
          setLocale('fr')
          frRef.current?.focus()
        } else if (e.key === 'ArrowLeft') {
          e.preventDefault()
          setLocale('en')
          enRef.current?.focus()
        }
      }

      return (
        <div
          role="tablist"
          aria-label="Content language"
          className="jeeby-cms-locale-switcher"
        >
          <button
            ref={enRef}
            type="button"
            role="tab"
            aria-selected={locale === 'en'}
            tabIndex={locale === 'en' ? 0 : -1}
            onClick={() => setLocale('en')}
            onKeyDown={handleKeyDown}
            className="jeeby-cms-locale-tab"
          >
            EN
          </button>
          <button
            ref={frRef}
            type="button"
            role="tab"
            aria-selected={locale === 'fr'}
            tabIndex={locale === 'fr' ? 0 : -1}
            onClick={() => setLocale('fr')}
            onKeyDown={handleKeyDown}
            className="jeeby-cms-locale-tab"
          >
            FR
          </button>
        </div>
      )
    }
    ```
