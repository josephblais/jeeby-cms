<task type="auto">
  <name>Task 1: Install devDependencies and run build</name>
  <files>package-lock.json</files>
  <action>
Run npm install to install tsup (the only devDependency declared in package.json):

```bash
npm install
```

Then run the build:

```bash
npm run build
```

The build script is `tsup && cp styles/cms.css dist/styles.css`. It will:
1. Run tsup, which reads tsup.config.js and builds all three entries (index, admin, server) in both ESM and CJS format
2. Copy styles/cms.css to dist/styles.css

Expected output files after build:
- dist/index.mjs (ESM client — starts with "use client")
- dist/index.js (CJS client — starts with "use client")
- dist/admin.mjs (ESM admin)
- dist/admin.js (CJS admin)
- dist/server.mjs (ESM server — NO "use client")
- dist/server.js (CJS server — NO "use client")
- dist/styles.css (copied from styles/cms.css)

If the build fails, diagnose by checking:
1. Is tsup installed? Run `ls node_modules/.bin/tsup`
2. Is tsup.config.js valid? Run `node -e "import('./tsup.config.js').then(m=>console.log('valid',Array.isArray(m.default)?m.default.length:'not array')).catch(console.error)"`
3. Do all source entry files exist? Run `ls src/index.js src/admin/index.js src/server/index.js styles/cms.css`

Common issue — outExtension mismatch: After first build, verify actual extensions with `ls dist/`. If tsup produced .cjs instead of .js for CJS output, update the outExtension function and the package.json exports field to match. The exports field in package.json must exactly match the file extensions tsup produces.

<task type="auto">
  <name>Task 2: Run full verification suite</name>
  <files></files>
  <action>
Run the full verification script created in Plan 02:

```bash
node scripts/verify-exports.js
```

The script checks:
- All 6 dist JS files exist (PKG-02)
- dist/styles.css exists (PKG-03)
- dist/index.mjs and dist/index.js start with "use client" (PKG-02 banner)
- dist/server.js and dist/server.mjs do NOT contain "use client" (PKG-02 server safety)
- dist/index.mjs does not contain "firebase/app" or "framer-motion/dist" (PKG-04 peer deps external)
- dist/index.js exports CMSProvider, Blocks, Block, useCMSContent (PKG-01)
- dist/admin.js exports AdminPanel, withCMSAuth (PKG-01)
- dist/server.js exports getCMSContent (PKG-01)

If any check fails, the script prints which check failed and a descriptive error. Common failures and fixes:

FAIL "dist/index.mjs starts with use client":
- The banner in tsup.config.js may be wrong. Check that the first config block has: banner: { js: '"use client";' }
- Note the nested quotes: the JS string is "use client"; — so in JSON/JS you need '"use client";' (single quotes wrapping double quotes)

FAIL "dist/server.js does NOT contain use client":
- The server config block in tsup.config.js has a banner that shouldn't be there. Remove any banner from the server config block.

FAIL "dist/index.js is missing export: CMSProvider":
- TSUP may have produced .cjs instead of .js for CJS output. Check actual filenames with `ls dist/`. If .cjs, update outExtension and package.json exports.

FAIL "dist/index.mjs does not bundle firebase":
- The external array in tsup.config.js is missing 'firebase'. Add it.

Once all checks pass, the script prints "Build verification PASSED. Phase 1 scaffolding is complete." and exits 0.
