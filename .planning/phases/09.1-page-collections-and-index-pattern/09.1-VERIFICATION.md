### Human Verification Required

#### 1. CreatePageModal prefix-locked slug input

**Test:** In the admin panel, create a Collection page (e.g. "blog"). Then click "New Page", change type to "Page", and select "blog" as the parent collection.
**Expected:** The slug input renders as a two-part control: a locked, visually distinct `/{parentSlug}/` prefix span followed by an editable leaf input. The slug hint below shows "Full path: /blog/my-slug". The prefix is not focusable or editable.
**Why human:** CSS layout (flex, prefix styling) and `aria-hidden` prefix behavior require browser rendering to verify.

#### 2. PageManager collection expand/collapse

**Test:** With collection and entry pages in Firestore, open the Page Manager. Verify a COLLECTIONS section header appears above collection rows, each with a toggle button. Click the toggle.
**Expected:** Entry rows under the collection reveal/hide. The toggle button's `aria-expanded` attribute toggles between true/false. The chevron SVG rotates 90 degrees (CSS `data-expanded="true"` transition). Keyboard: Enter/Space on the toggle also works.
**Why human:** Set-based expand state, CSS transition, and keyboard interaction require live browser testing.

#### 3. Delete guard for collections with entries

**Test:** In PageManager, attempt to delete a Collection that has at least one Entry child.
**Expected:** An inline error alert appears above the table: "This collection has N entries. Delete or reassign them first." The DeletePageModal does NOT open. A Dismiss button clears the error.
**Why human:** Requires a live Firestore connection to return children from `getCollectionPages` in `handleDeleteClick`.
