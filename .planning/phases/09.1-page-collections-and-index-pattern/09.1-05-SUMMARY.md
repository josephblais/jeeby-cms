## Decisions Made

- Entry rows are read-only in this phase (no Rename/Delete) — editors manage entries via the full editor page. Intentional scope reduction per plan.
- Each collection tbody carries `id=cms-collection-entries-{slug}` so the toggle button's `aria-controls` has a valid, unique DOM target per WCAG 4.1.2.
- The D-20 error message uses a ternary (`count === 1 ? 'entry' : 'entries'`) where the plural branch is a template literal so the literal text `entries. Delete or reassign them first.` appears in source for the source-scan test.

## Deviations from Plan

None - plan executed exactly as written, including the minor fix to the D-20 message structure to ensure the literal `entries. Delete or reassign them first.` string appeared in source for the test assertion.

## Issues Encountered

Task 3 first attempt: The JSX pattern `{...? 'entry' : 'entries'}. Delete or...` placed `'}` between `entries` and `.`, so the source-scan test regex `/entries\. Delete or reassign them first/` did not match. Fixed by restructuring to a ternary that produces separate string literals for singular and plural — the plural template literal contains the required literal text.

## Known Stubs

None. Entry rows are intentionally read-only per plan scope (not a data stub — editors manage entries via the full editor). Future plans may add inline actions on entry rows if needed.
