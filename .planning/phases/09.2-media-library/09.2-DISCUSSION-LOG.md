## Metadata prompt timing

**Question:** After an image finishes uploading to the media library, when does the title/alt form appear?

| Option | Description |
|--------|-------------|
| Inline replace | Upload card flips to title/alt form inline in the grid |
| Sheet/overlay step | Small overlay appears over the modal asking for title/alt |
| At selection time | Image lands in library immediately; prompt appears at selection |

**Selected:** Inline replace — upload card flips to title/alt form inline

---

**Question:** Is alt text required before 'Save to Library', or can it be left blank?

| Option | Description |
|--------|-------------|
| Required | Can't save until alt text filled |
| Optional with warning | Can save blank but warning appears |
| Optional, no warning | Blank silently accepted |

**Selected:** Optional with warning

---

## Auto-populate behavior

**Question:** When a library image is selected into a block, what happens to the block's existing alt text?

| Option | Description |
|--------|-------------|
| Always replace | Library alt always overwrites block alt |
| Fill empty only | Only fills empty block alt |
| Prompt if conflict | Ask before overwriting existing value |

**Selected:** Prompt if conflict

---

**Question:** Where does the conflict prompt appear?

| Option | Description |
|--------|-------------|
| Inline in block editor | Small inline prompt in the editor after modal closes |
| Inside the media modal | Before modal closes |
| Always replace, no prompt | Library alt always wins |

**Selected:** Inline in block editor — "Library alt: '[value]'" with [Keep mine] / [Replace] buttons

## Home page upload button

**Question:** What does 'Upload Media' on the home admin page do?

| Option | Description |
|--------|-------------|
| Opens full library modal | Same modal, browse+upload mode, no selection callback |
| Upload-only panel | Focused drop zone + form, no browsing |

**Selected:** Opens full library modal in browse+upload mode

## GalleryEditor selection

**Question:** When GalleryEditor opens the library in multi-select mode, how does selection work?

| Option | Description |
|--------|-------------|
| Checkboxes + confirm | Checkboxes on hover/focus, "Add N images" confirm button |
| Click-to-toggle, running count | Click toggles highlight, sticky footer with count |
| Single-select only | Same as ImageEditor, select one at a time |

**Selected:** Checkboxes + confirm — "Add N images" sticky footer

## Firestore schema

**Question:** Where should media metadata live in Firestore?

| Option | Description |
|--------|-------------|
| /cms/media collection | Separate collection, pageable, fields: storageUrl, storagePath, title, alt, uploadedAt, mimeType, size |
| /cms/config/media array | Single config doc array — hits 1MB limit at scale |

**Selected:** /cms/media collection with full field set

---

## Modal dimensions

**Question:** How large should the media library modal be?

| Option | Description |
|--------|-------------|
| Large centered card | ~90vw/85vh, backdrop visible, consistent with existing modals |
| Full-screen overlay | Full viewport, more grid space |

**Selected:** Large centered card (~90vw/85vh)

## Library-only vs block upload

**Question:** Do ImageEditor and GalleryEditor keep their existing direct-upload buttons?

| Option | Description |
|--------|-------------|
| Keep both | Existing Upload + new Select from Library |
| Library only | Remove direct upload, all via library |
| Library-first, direct fallback | Library is primary, old upload secondary |

**Selected:** Keep both buttons — but existing Upload button now also creates a library entry + prompts for title/alt inline in the editor

---

## Edit after upload

**Question:** Can editors update a media item's title/alt after it's saved to the library?

| Option | Description |
|--------|-------------|
| In library modal only | Click image in grid → inline edit panel → save back to Firestore |
| Not in v1 | Write-once for now |
| Anywhere the image appears | Edit in block also offers to update library — complex |

**Selected:** In the library modal only — click image → inline edit → save to Firestore
