Declared values (multiples of 4 only). Source: established in Phase 8, confirmed in `styles/cms.css`.

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Button padding-y, field gap, progress bar margin |
| md | 16px | Default element spacing, modal padding inner, grid gap between cards |
| lg | 24px | Modal header padding, section breaks within the library panel |

Exceptions:
- Modal card padding for the standard ModalShell: 24px (existing `.jeeby-cms-modal-card` value — do not change).

- Sticky multi-select footer: 12px vertical padding, 16px horizontal, fixed to card bottom edge.

## Interaction States

### Media Card States (state machine)

Each card in the library grid passes through these states. Transitions animated by Framer Motion `AnimatePresence mode="wait"` with `duration: 0.2s` (or `0.01s` when `useReducedMotion()` is true).

| State | Visual | User Action to Enter |
|-------|--------|---------------------|
| `idle` | Thumbnail image, hover reveals checkbox in multi-select mode | Default |
| `uploading` | Progress bar over thumbnail slot, status text below | File selected from upload button |
| `pending-meta` | Title + alt form animates in over the card area | Upload resolves |
| `saved` | Thumbnail visible, brief success indicator (optional) | User clicks "Save to Library" |
| `edit` | Title + alt form re-appears over saved thumbnail | User clicks the card to edit |

### Infinite Scroll

- Sentinel `div` after last card, `aria-hidden="true"`.
- `IntersectionObserver` with `rootMargin: '100px'` pre-fetches next batch before sentinel reaches viewport.
- While loading next batch: a single row of skeleton cards appends below existing items.
- When `hasMore: false`: sentinel removed; no "end of list" label required.

### Multi-Select (GalleryEditor mode)

- Checkboxes visible on hover and `:focus-within` on each card.
- Checked state: `aria-checked="true"` on checkbox input + `.jeeby-cms-media-card--selected` modifier class.
- Sticky footer appears when `selected.size > 0`. Disappears when `selected.size === 0`.
- Footer copy: "{N} selected" + "Add {N} image" / "Add {N} images" button.
- Clicking "Add {N} images": calls `onSelect(selectedItems)`, modal closes, focus returns to `triggerRef`.
- Selection state lives inside the modal component (not parent). Resets on each `open: true` transition.

### Close-with-pending-meta guard

If user attempts to close the modal while any card is in `pending-meta` state (upload complete, metadata not yet saved), show a warning: a `window.confirm`-style check is explicitly excluded from this design system. Instead: disable the backdrop click-to-close and show an inline banner inside the modal header: "Finish saving the upload details before closing." The close button becomes disabled. Once all `pending-meta` cards are either saved or explicitly discarded, close is re-enabled.

Note: "Discard" option on a pending-meta card: a small "Discard" ghost link below the form. Clicking it removes the card from the grid (the Storage file is orphaned — accepted v1 tradeoff, documented in RESEARCH.md Pitfall 2).

## Accessibility Contract

These requirements apply to all new components in this phase. They are non-negotiable per CLAUDE.md.

### Focus management

| Trigger | Focus target |
|---------|-------------|
| Modal opens | First focusable element in modal (ModalShell handles this automatically) |
| Modal closes (any reason) | `triggerRef.current` — the button that opened the modal |
| Upload completes → card enters `pending-meta` | First input in the metadata form (title input) |
| "Save to Library" succeeds | Next focusable card or the upload button if no next card |
| Alt conflict prompt appears in block editor | `keepBtnRef.current` — the "Keep mine" button |
| Alt conflict resolves | URL input in ImageEditor |

### ARIA

| Element | Role / Properties |
|---------|------------------|
| Modal | `role="dialog"` via ModalShell; `aria-labelledby="media-library-heading"` |
| Modal h2 | `id="media-library-heading"` |
| Infinite scroll sentinel | `aria-hidden="true"` |
| Upload progress | `aria-live="polite"` region; progress text announces percentage or "Upload complete" |
| Alt blank warning | `role="alert"` (injected content, must announce immediately) |
| Alt conflict prompt | Focus moved to first button — no `role="alert"` needed (focus is the announcement) |
| Card checkboxes | `<input type="checkbox">` with `aria-label="{title or filename}"` |
| Multi-select footer | `role="region"` with `aria-label="Selection summary"` |
| Pending-meta close guard banner | `role="alert"` |
| Media card "Edit details" trigger | `<button aria-label="Edit details for {title}">` |
| Skeleton cards | `aria-hidden="true"` |

### Keyboard

- All cards reachable via Tab in grid order.
- In multi-select mode: Space bar toggles checkbox on focused card.
- Enter on a card in single-select mode: selects and closes modal (same as click).
- Escape: closes modal if no pending-meta cards (standard ModalShell behavior); suppressed while pending-meta guard is active.
- Sticky footer "Add N images" button: Tab-reachable after all card checkboxes.

## Animation Contract

Source: CONTEXT.md "specifics"; RESEARCH.md Pattern 4; CLAUDE.md "animation: subtle, purposeful."

| Interaction | Animation | Reduced motion fallback |
|-------------|-----------|------------------------|
| Modal open/close | ModalShell handles (existing Framer Motion scale+opacity) | ModalShell already respects `useReducedMotion()` |
| Card pending-meta crossfade | `AnimatePresence mode="wait"` opacity 0→1, duration 0.2s | `duration: 0.01` when `useReducedMotion()` is true |
| Card saved crossfade (back to thumbnail) | Same as above | Same |
| Sticky footer enter/exit | `AnimatePresence` opacity + translateY(8px→0), duration 0.15s | `duration: 0.01` |
| Skeleton pulse | `@keyframes jeeby-pulse` (add to cms.css); opacity 0.4→0.8 loop | `@media (prefers-reduced-motion: reduce)` disables animation |
| Upload progress bar | CSS `transition: width 120ms linear` (existing pattern from Phase 9) | None needed — width change is functional, not decorative |

No new animations are purely decorative. Each animation either communicates state change or orients the user spatially.
