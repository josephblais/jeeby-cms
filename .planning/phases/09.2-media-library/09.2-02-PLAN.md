**State management:**
- `items` (array) — loaded media items from Firestore
- `cursor` (ref, NOT state — Firestore DocumentSnapshot, see RESEARCH anti-pattern) — next page cursor
- `hasMore` (boolean) — whether more pages exist
- `isLoading` (boolean) — initial load in progress
- `isLoadingMore` (boolean) — loading next page
- `error` (string | null) — Firestore fetch error
- `selected` (Set) — selected item IDs in multi-select mode; lives INSIDE the modal (per RESEARCH Pitfall 7)
- `pendingUploads` (Map<tempId, {file, progress, storageUrl, storagePath, state}>) — tracks upload state per file
  - state: 'uploading' | 'pending-meta' | 'saved'
- `closeGuardActive` (boolean) — derived: true when any pendingUpload is in 'pending-meta' state
- `cursorRef` (ref) — `const cursorRef = useRef(null)` — holds the Firestore DocumentSnapshot cursor for pagination (NOT state — prevents stale closure issues per RESEARCH anti-pattern)

**Reset on open:** Use `useEffect` watching `open` — when `open` becomes true, reset `selected` to empty Set, reset `items` to [], reset `cursorRef.current = null`, call `fetchInitialPage()`.

**fetchInitialPage function:**
```javascript
const fetchInitialPage = useCallback(async () => {
  setIsLoading(true)
  setError(null)
  try {
    const result = await listMediaPaginated(db)
    setItems(result.items)
    cursorRef.current = result.nextCursor
    setHasMore(result.hasMore)
  } catch (err) {
    console.error('[jeeby-cms] Failed to load media library:', err)
    setError('Could not load your media library. Check your connection and try again.')
  } finally {
    setIsLoading(false)
  }
}, [db])
```

**fetchNextPage function (for infinite scroll):**
```javascript
const fetchNextPage = useCallback(async () => {
  if (!hasMore || isLoadingMore) return
  setIsLoadingMore(true)
  try {
    const result = await listMediaPaginated(db, { cursor: cursorRef.current })
    setItems(prev => [...prev, ...result.items])
    cursorRef.current = result.nextCursor
    setHasMore(result.hasMore)
  } catch (err) {
    console.error('[jeeby-cms] Failed to load more media:', err)
  } finally {
    setIsLoadingMore(false)
  }
}, [db, hasMore, isLoadingMore])
```

**IntersectionObserver for infinite scroll (per RESEARCH Pattern 3):**
```javascript
const sentinelRef = useRef(null)
useEffect(() => {
  if (!hasMore || isLoadingMore || isLoading) return
  const el = sentinelRef.current
  if (!el) return
  const observer = new IntersectionObserver(
    ([entry]) => { if (entry.isIntersecting) fetchNextPage() },
    { rootMargin: '100px' }
  )
  observer.observe(el)
  return () => observer.unobserve(el)
}, [hasMore, isLoadingMore, isLoading, fetchNextPage])
```

**Upload handling (per D-05, D-06, D-08, D-16):**
- File input accepts `image/jpeg,image/png,image/gif,image/webp`, `multiple` enabled
- For each selected file: validate with `validateImageFile`, generate temp ID, add to `pendingUploads` map in 'uploading' state
- Upload each file via `uploadFile(storage, file, path, onProgress, cancelRef)`
- On completion: transition card state to 'pending-meta', store `storageUrl` and `storagePath`
- Card flip: AnimatePresence mode="wait" crossfade (duration 0.2s, reduced motion 0.01s)
- Pending-meta form shows: title input (no warning), alt input (warning if blank per D-06), "Save to Library" button, "Discard upload" ghost link
- "Save to Library" calls `addMediaItem(db, { storageUrl, storagePath, title, alt, mimeType: file.type, size: file.size })`
- On save success: add item to `items` array, remove from `pendingUploads`, transition card to show saved thumbnail

**Close guard (per UI-SPEC interaction states):**
- Derive `closeGuardActive` from pendingUploads: `[...pendingUploads.values()].some(u => u.state === 'pending-meta')`
- When `closeGuardActive`:
  - Disable the close button (add `disabled` attribute)
  - Show inline banner: `role="alert"` "Finish saving the upload details before closing."
  - Override `onClose` prop to no-op (do not call parent onClose)
  - ModalShell's Escape key handler calls onClose — which is now a guarded function that checks `closeGuardActive` first

**Browse mode (per D-13):**

**Single-select mode (per D-14):**
- No checkboxes, no sticky footer
- Clicking a saved card calls `onSelect(item)` where `item = { storageUrl, alt, title, id }`
- Modal closes automatically after selection

**Multi-select mode (per D-17):**
- Each card shows a checkbox (visible on hover/focus, always visible when checked)
- Checkbox: `<input type="checkbox" aria-label="{title or 'Untitled image'}" />`
- `selected` state: `Set<id>` — toggled on checkbox change
- Sticky footer visible when `selected.size > 0`:

- Card flip to title/alt form (same animation as pending-meta)
